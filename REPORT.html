04Bugs đã gặp và cách giải quyết

Bug 1 — Gemma list-content crash

Root causeModel Gemma trả về AIMessage.content dạng list[dict] thay vì str. Code gốc gọi result.content.strip() → AttributeError: 'list' object has no attribute 'strip'.
python — common/llm.pydef content_to_str(raw) -> str:
    if isinstance(raw, str):
        return raw
    if isinstance(raw, list):
        return "".join(
            block.get("text", "") if isinstance(block, dict) else str(block)
            for block in raw
        )
    return str(raw) if raw is not None else ""

Áp dụng tại 5 file: law_agent, tax_agent, compliance_agent, customer_agent, stage_4. Đây là fix quan trọng nhất của dự án vì nó ảnh hưởng toàn bộ output pipeline.

Tối ưu 1 — Merge analyze_law + check_routing

Thay vì 2 LLM calls tuần tự, dùng 1 call trả về JSON có cả law_analysis và routing decision:

python — law_agent/graph.pyasync def analyze_and_route(state: LawState) -> dict:
    # 1 LLM call thay vì 2, tiết kiệm ~15s
    messages = [
        SystemMessage(content=(
            "You are a corporate litigation attorney. Respond in the same language as the question.\n"
            'Reply ONLY with valid JSON: {"law_analysis":"...","needs_tax":bool,"needs_compliance":bool}'
        )),
        HumanMessage(content=state["question"]),
    ]
    result = await llm.ainvoke(messages)
    parsed = json.loads(content_to_str(result.content))
    return {"law_analysis": parsed["law_analysis"],
            "needs_tax": parsed["needs_tax"],
            "needs_compliance": parsed["needs_compliance"]}

Tối ưu 2 — Bỏ LLM trong aggregate()

Thay vì dùng LLM synthesis (~20s), ghép các section bằng template Markdown:

python — law_agent/graph.pyasync def aggregate(state: LawState) -> dict:
    # Không LLM — tiết kiệm ~20s
    sections = []
    if state.get("law_analysis"):
        sections.append(f"## Legal Analysis\n\n{state['law_analysis']}")
    if state.get("tax_result"):
        sections.append(f"## Tax Analysis\n\n{state['tax_result']}")
    if state.get("compliance_result"):
        sections.append(f"## Regulatory Compliance\n\n{state['compliance_result']}")
    return {"final_answer": "\n\n---\n\n".join(sections)}

07Bài học rút ra

🔬 Kỹ thuật

  • Luôn normalize LLM output — không assume content là str. Viết helper content_to_str() và áp dụng tại tất cả điểm tiếp xúc với LLM response. Các model khác nhau (Gemma, Qwen, Claude) có format output khác nhau.
  • create_react_agent không có tools = lãng phí — React agent luôn chạy ≥2 LLM turns (generate → check tools). Nếu không có tools, dùng ainvoke trực tiếp tiết kiệm ~50% thời gian mỗi agent.
  • LLM call là bottleneck chính với local models — Mỗi call ~10–20s. Giảm số LLM calls (merge, bỏ synthesis) hiệu quả hơn nhiều so với tối ưu network hay code Python.
  • max_tokens là lever quan trọng nhất — Với Ollama local, model sinh ~20–40 tokens/s. Giới hạn output tokens = giới hạn thời gian trực tiếp. 600 tokens ≈ 15–30s tuỳ hardware.
  • System prompt ngắn ≠ chất lượng kém — Prompt 3 dòng focused cho kết quả ngang prompt 15 dòng verbose, nhưng fast hơn đáng kể vì ít token context phải xử lý.
  • Thêm "Respond in same language as question" — Một dòng đơn giản giúp model tự xử lý ngôn ngữ, không cần detect hay translate thủ công.

🏗 Kiến trúc

  • A2A protocol JSON-RPC — Method name phải chính xác. Đọc source code SDK (.venv/Lib/site-packages/a2a/types.py) là cách nhanh nhất để tìm đúng format, đừng đoán.
  • CORS phải cấu hình ngay từ đầu — Nếu có client HTML/browser sẽ gọi vào services, thêm CORSMiddleware từ lúc thiết kế, không phải debug sau.
  • LangGraph Send API cho parallel dispatch — Trả về list[Send] từ conditional edge function là cách đúng để fan-out parallel. Không dùng asyncio.gather bên ngoài graph.
  • Registry service discovery — Thêm retry logic khi register (10 lần, delay 2s) vì Registry có thể chưa sẵn sàng khi leaf agents startup. Thứ tự khởi động quan trọng.

🛠 DevOps & Tooling

  • .gitignore phải làm TRƯỚC khi git add — Model binary, .venv/, .env phải trong .gitignore trước lần commit đầu tiên. Xóa sau bằng git rm --cached hoạt động được nhưng mất công.
  • uv thay pip — Trong dự án dùng uv, phải dùng uv pip install hoặc uv run, không phải pip install hay python -m trực tiếp.
  • PowerShell version matters — Syntax ?., ??, ternary ? : chỉ có trong PS 7+. Windows mặc định PS 5.1. Script production nên target PS 5.1 để không cần user cài thêm.
  • Fallback strategy cho API key — Thiết kế get_llm() với priority: env flag → API key có → local fallback. Khi quota cạn, flip 1 biến môi trường là đủ, không cần sửa code.

💡 Process

  • Test end-to-end sớm — Chạy test_client.py ngay sau khi có 5 services để xác nhận pipeline, không đợi đến khi UI xong.
  • Debug từ ngoài vào — Khi pipeline fail: kiểm tra health /.well-known/agent.json trước, sau đó log, rồi mới code. Đừng sửa code khi chưa biết service có online không.
  • Visualization giúp debug rất nhiều — Dashboard SVG với live log và timing giúp thấy ngay bottleneck (t-spec: 75s, t-analyze: 5s → rõ ràng Tax/Compliance là vấn đề) mà không cần đọc logs terminal.
